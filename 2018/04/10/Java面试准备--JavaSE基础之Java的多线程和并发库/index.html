Java面试准备--JavaSE基础之Java的多线程和并发库

JavaSE基础

Java的多线程和并发库

  1. 传统使用Thread和Runnable接口实现
    在Thread子类覆盖的run方法中编写运行代码

  2. 在传递给Thread对象的Runnable对象的run方法中编写代码

  3. 定时器Timer和TimerTask
    面试题：使用定时器,间隔 4 秒执行一次，再间隔 2 秒执行一次，以此类推执行

  4. 线程的互斥与同步
    线程同步的主要任务是使并发执行的各线程之间能够有效的共享资源和相互合作，从而使线程的执行具有可再现性。
    线程并发执行，由于共享资源和线程协作，使线程之间会存在两种制约关系：

    1. 间接相互制约
2. 直接相互制约

    间接相互制约可以称为互斥，直接相互制约可以称为同步。也可以这样说，同步就是互斥，互斥是一种特殊的同步。
    面试题：子线程运行执行 10 次后，主线程再运行 5 次。这样交替执行三遍

  5. 多线程共享数据
    Java传统线程共享数据的方式有两种：

    1. 多个线程行为一致，操作同一个数据源，可以使用同一个Runnable对象，这个Runnable对象中有那个共享数据，例如买票系统
2. 多个线程行为不一致，操作同一个数据源，这就有不同的Runnable对象，例如银行存取款。
