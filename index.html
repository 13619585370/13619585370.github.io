  1. HashMap 排序题（面试题）
    已知一个 HashMap集合， User 有 name（String）和 age（int）属性。请写一个方法实现对
    HashMap 的排序功能，该方法接收 HashMap为形参，返回类型为 HashMap，
    要求对 HashMap 中的 User 的 age 倒序进行排序。排序时 key=value 键值对不得拆散。
    解题思路：因为HashMap本身就是不可以排序的，但是他的子类LinkedHashMap的结构是有序的所以

    （1）、首先我们获取到Map集合的键值对存储到Set集合中。
（2）、将Set集合转化为List集合。
（3）、可以使用工具类Collections中的排序方法进行排序
（4）、排序之后在存储到一个新的有序的LinkedHashMap集合中

  2. 集合的安全性问题
    ArrayLsit、HashSet、HashMap是线程安全的吗？不是线程安全的
    在集合中，Vector和HashTable是线程安全的，他们都是在相关核心的方法上加上了synchronized 关键字。
    Collections 工具类提供了相关的 API，可以让上面那 3 个不安全的集合变为安全的。

    实现原理就是在集合的核心方法上加上synchronized 关键字。

  1. 传统使用Thread和Runnable接口实现
    在Thread子类覆盖的run方法中编写运行代码

  2. 在传递给Thread对象的Runnable对象的run方法中编写代码

  3. 定时器Timer和TimerTask
    面试题：使用定时器,间隔 4 秒执行一次，再间隔 2 秒执行一次，以此类推执行

  4. 线程的互斥与同步
    线程同步的主要任务是使并发执行的各线程之间能够有效的共享资源和相互合作，从而使线程的执行具有可再现性。
    线程并发执行，由于共享资源和线程协作，使线程之间会存在两种制约关系：

    1. 间接相互制约
2. 直接相互制约

    间接相互制约可以称为互斥，直接相互制约可以称为同步。也可以这样说，同步就是互斥，互斥是一种特殊的同步。
    面试题：子线程运行执行 10 次后，主线程再运行 5 次。这样交替执行三遍

  5. 多线程共享数据
    Java传统线程共享数据的方式有两种：

    1. 多个线程行为一致，操作同一个数据源，可以使用同一个Runnable对象，这个Runnable对象中有那个共享数据，例如买票系统
2. 多个线程行为不一致，操作同一个数据源，这就有不同的Runnable对象，例如银行存取款。

Java web 的深入分析

Java Web 中的请求

服务架构

  1. C/S 架构
    C/S架构就是我们常见的客户端/服务器架构，优点就是能够充分发挥客户端PC的处理能力，很多工作都可以在客户端进行处理
    然后提交到服务器端，客户端响应速度快，具体优点有：
    （1）、应用服务器运行数据负荷较轻。
    （2）、数据的存储管理功能较为透明。

  2. B/S架构
    B/S架构就是浏览器/服务器架构。通常服务端都是基于HTTP请求的，基于Http的服务器有很多：Apache、IIS、Nginx、Tomcat、
    JBoss等。
    HTTP采用无状态的短连接的通信方式，通常情况下，一次请求就完成一次数据交互，也对应一个业务逻辑。通讯完成之后就会断开
    连接，不可能一直连接下去。

    解析URL地址的过程：当用户输入一个url的时候，首先是DNS会将这个url解析成相应的IP地址，然后会根据这个IP地址找到相应的
    服务器，向这个服务器提交一个get请求，有这个服务器决定返回默认的数据资源给这个用户。
